## Cambios principales realizados

### 1) Ajustes de rutas y canonicas
- Se normalizaron URLs para despliegue en raiz con archivos `.html`:
  - `https://walkdogentrena.com/landing-1.html`
  - `https://walkdogentrena.com/landing-2.html`

  - `canonical`
  - `og:url`
- Se evitaron rutas tipo carpeta (`/landing-1/` o `/landing-2/`) cuando no existen en servidor.

### 2) Open Graph y Twitter meta tags
- Se agregaron y ajustaron etiquetas OG/Twitter en ambas landings:
  - `og:type`, `og:url`, `og:title`, `og:description`
  - `og:image`, `og:image:secure_url`, `og:image:type`, `og:image:alt`

- Se definieron descripciones distintas para cada landing.

### 3) Compatibilidad de imagen para previews
- Se detectaron errores en Facebook Debugger para `landing-2` (imagen OG no procesable / error desconocido).

- Resultado final:
  - `landing-1` usa `assets/og-share.jpg`
  - `landing-2` usa `assets/og-share-2.png`
- Se incluyeron dimensiones OG:
  - `og:image:width = 1536`
  - `og:image:height = 1024`

### 4) Optimizacion de peso de imagenes OG
- `og-share.jpg` fue re-codificada y reducida de tamano.
- `og-share-2.jpg` fue re-codificada para compatibilidad.

- Facebook Debugger devolvio combinaciones de:
  - timeout cURL
  - codigo 418
  - `og:url` antigua en cache
  - error de lectura de `og:image`
- Diferencia entre estado local y estado servido por hosting en algunos momentos.

## Solucion final validada
- El usuario confirma que ya funciona la vista previa.

  - `landing-2.html` -> OG con `assets/og-share-2.png`

## Nota de mantenimiento
Si vuelve a fallar una preview en redes:
1. Confirmar que la URL existe y responde 200/206.
2. Confirmar que `og:url` coincide exactamente con la URL depurada.
3. Probar imagen OG en JPG o PNG (preferir PNG cuando haya errores raros de scraper).
4. Revisar firewall/WAF (Wordfence, Cloudflare, hosting) para bots de Facebook.
