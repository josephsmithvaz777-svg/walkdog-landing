## 3. Video responsive (iteraciones del día)

### Variables CSS

```css
--video-target-vw: 78;
--video-max-px: 1400;
--video-vimeo-ratio-w: 5;
--video-vimeo-ratio-h: 4.15;   /* casi cuadrado */
--video-cinema-overscan: 1.06;
```

### Problema resuelto

El JS `lockVideoAspect` recortaba el ancho por `max-height-vh`, haciendo que cambios de `78vw` no se notaran.

**Solución:** `readVideoTargetWidth()` lee `--video-target-vw` del viewport; en modo cinema usa el ancho de `hero-video__frame`.
