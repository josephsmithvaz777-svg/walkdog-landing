## 1. Contexto: por qué Vimeo

- YouTube generaba bordes negros, UI nativa difícil de ocultar y limitaciones con la IFrame API.
- El cliente subió el vídeo a **Vimeo** (plan gratuito).
- En plan free **no se pueden ocultar** título, logo ni autor desde el panel de Vimeo; solo vía parámetros de embed (limitados).

### Configuración recomendada en Vimeo (panel Embed)

| Opción | Valor recomendado |
|--------|-------------------|
| **Size** | **Responsive** (no Fixed 1920×1080) — la landing controla el tamaño |
| **Where embed** | Anywhere |
| **Autoplay / Mute** | Opcional en panel; la landing fuerza `autoplay=1&muted=1` en la URL |

**No pegar** el embed crudo de Vimeo en WordPress además de la landing — provoca duplicados.

## 2. Problemas encontrados durante la implementación

| Problema | Causa |
|----------|--------|
| Vídeo **por detrás** e imagen **congelada** delante | Capas `video-play-gate`, `video-yt-shield`, iframe duplicado o recarga sin destruir player |
| **Doble vídeo** al pulsar «Activar sonido» | `new Vimeo.Player(iframe)` + recarga de `src` dejaba audio del player anterior |
| Mini reproductor flotante (PiP) | `picture-in-picture` en `allow` y recargas del iframe |
| **Franjas negras** laterales o arriba/abajo | Desajuste de ratio contenedor vs. vídeo; píxeles fraccionarios al 100% zoom; contenido 4:3 dentro de archivo 1920×1080 |
| A **80% zoom** se ve bien, a **100%** aparecen franjas | Contenedor con dimensiones no enteras (subpíxeles) |
| Contenedor **demasiado ancho** (16:9) | El cliente pidió formato más rectangular, no cinematográfico |

### Enfoques descartados

1. **Vimeo Player API** (`player.js` + `new Vimeo.Player`) — capas duplicadas y bug al desmutear.
2. **Recargar iframe** al desmutear — player anterior seguía en audio.
3. **Play-gate + shield** sobre el iframe — blur = imagen congelada en autoplay.
4. **`scale()` agresivo** combinado con recarga de player — empeoraba congelamiento visual.
5. **Contenedor Fixed 1920×1080** del embed de Vimeo — rompe diseño responsive.

## 3. Solución final (implementación actual)

### Arquitectura

```
video-embed-root (data-vimeo-id="1202982270")
├── .video-vimeo-mount
│   └── iframe.video-vimeo-iframe   ← UN solo iframe, estático en HTML
├── .video-progress                 ← barra visual + CTA pulse
└── .video-unmute-btn               ← activa sonido vía postMessage
```

**Principios clave:**

1. **Un solo iframe** en el HTML (JS no lo duplica).
2. **No se carga** `player.js` ni `new Vimeo.Player()`.
3. **Unmute** con **`postMessage`** al iframe.
4. Autoplay **`autoplay=1&muted=1`** en URL.
5. Contenedor Vimeo **5:3** (rectangular, más ancho que 4:3, menos que 16:9).
6. Iframe en modo **cover** para llenar el contenedor sin franjas arriba/abajo.
7. **`lockVideoAspect()`** fija ancho/alto en píxeles enteros (evita franjas al 100% zoom).

## 4. Formato visual del vídeo (evolución)

| Iteración | Ratio contenedor | Notas |
|-----------|------------------|-------|
| Inicial | 16:9 | Franjas laterales; muy ancho |
| v2 | 4:3 | Más rectangular; franjas arriba/abajo con vídeo 16:9 |
| v3 | 3:2 | Un poco más ancho (pedido del cliente) |
| **Final** | **5:3** | Equilibrio: rectangular pero con más presencia horizontal |

### Variables CSS actuales

```css
--video-vimeo-max-w: min(880px, 94vw);
--video-vimeo-ratio-w: 5;
--video-vimeo-ratio-h: 3;
```

### Cover del iframe (16:9 dentro de contenedor 5:3)

```css
.video-embed-root[data-vimeo-id] .video-vimeo-mount iframe {
  top: 50%;
  left: 50%;
  width: calc(100% * 16 / 9 * var(--video-vimeo-ratio-h) / var(--video-vimeo-ratio-w));
  height: 100%;
  transform: translate(-50%, -50%);
}
```

Con 5:3 el ancho del iframe ≈ **106.7%** del contenedor — rellena la altura y recorta ligeramente los lados sin franjas negras.

### Ajustar ancho en el futuro

Editar en `landing-1.css`:

| Más cuadrado | `--video-vimeo-ratio-w/h` | Ejemplo |
|--------------|---------------------------|---------|
| 4:3 | `4 / 3` | Más alto |
| 3:2 | `3 / 2` | Intermedio |
| **5:3** | **`5 / 3`** | **Actual** |
| 16:10 | `16 / 10` | Más ancho |
| 16:9 | `16 / 9` | Cinematográfico |

### Parámetros del embed

| Parámetro | Valor | Propósito |
|-----------|-------|-----------|
| `player_id`, `app_id` | `0`, `58479` | Del embed oficial Vimeo |
| `autoplay` | `1` | Inicia solo al cargar |
| `muted` | `1` | Requerido para autoplay |
| `controls` | `0` | Ocultar controles (parcial en plan free) |
| `title`, `byline`, `portrait` | `0` | Ocultar metadatos |
| `badge` | `0` | Ocultar logo Vimeo |
| `pip` | `0` | Sin picture-in-picture |
| `playsinline` | `1` | Inline en móvil |

## 6. JavaScript — `landing-drive-video.js`

### Detección de proveedor

```javascript
function mountRoot(root) {
  if (getVimeoId(root)) { mountVimeoRoot(root); return; }
  if (getYoutubeId(root)) { mountYoutubeRoot(root); return; }
  mountDriveRoot(root);
}
```

### Vimeo: progreso + unmute + aspect ratio

```javascript
function postVimeoCommand(iframe, method, value) {
  var payload = { method: method };
  if (value !== undefined) payload.value = value;
  iframe.contentWindow.postMessage(JSON.stringify(payload), 'https://player.vimeo.com');
}

function mountVimeoRoot(root) {
  if (root._wdVimeoMounted) return;
  root._wdVimeoMounted = true;
  lockSurface(root);
  scheduleProgressStart(root);
  lockVideoAspect(root);  // píxeles enteros + ratio 5:3 para Vimeo
  // … wire unmute btn → postVimeoCommand setVolume / setMuted / play
}
```

### `lockVideoAspect()` — Vimeo vs YouTube

| Proveedor | Ratio | Snap ancho | Cálculo alto |
|-----------|-------|------------|--------------|
| **Vimeo** | **5:3** | múltiplo de **4** | `h = snapped × 3 / 5` |
| YouTube | 16:9 | múltiplo de **16** | `h = snapped × 9 / 16` |

Fija `width`, `height`, `maxWidth` en px y centra con `margin: auto`.

### Flujo de audio

1. Al cargar: autoplay **silenciado** (`muted=1`).
2. ~1,2 s después: botón **«Activar sonido»**.
3. Clic → `postMessage`: `setVolume(1)` + `setMuted(false)` + `play`.
4. Botón oculto; hint → *«Audio activado.»*

**Nota:** autoplay con sonido sin clic **no es posible** por política de navegadores.
